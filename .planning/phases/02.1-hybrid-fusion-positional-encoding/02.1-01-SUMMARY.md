## Decisions Made

**1. Update existing functions instead of creating separate hybrid functions**
- Rationale: Single cache format simplifies pipeline. Old notebooks ignore the extra global_feat key in cached .pt files, new notebooks use it. No duplicate feature extraction runs needed.

**2. Normalized 2D coordinates (2 dims) over sinusoidal PE (64+ dims)**
- Rationale: 2560 train samples is small. Sinusoidal PE adds 64+ dimensions with higher expressiveness but risks overfitting. Normalized (row, col) in [0, 1] provides spatial awareness with minimal parameter overhead.

**3. Store global_feat as [1, 2048] not [2048]**
- Rationale: PyG default batching concatenates along dim=0. Storing as 1D [2048] would flatten to [batch_size * 2048]. Storing as 2D [1, 2048] produces correct [batch_size, 2048] after batching.

**4. Direct concatenation fusion without projection**
- Rationale: Research recommendation for small datasets: avoid unnecessary parameters. Direct concat [gnn_embed:128 + cnn_global:2048] -> Linear(2176, 16) is simpler than adding projection layers like Linear(2048, 256).

**5. Keep GraphSAGEClassifier unchanged**
- Rationale: Baseline model needed for comparison experiments. HybridGraphSAGE adds hybrid capability without disrupting existing experiments.
