  <name>Task 2: Add positional encoding and hybrid graph construction to graph.py</name>
  <files>src/graph.py</files>
  <action>
Add three new functions to src/graph.py. Keep all existing functions untouched.

**`add_positional_encoding_2d(x: torch.Tensor, grid_h: int = 7, grid_w: int = 7) -> torch.Tensor`**:
- Appends normalized (row, col) coordinates to node features.
- Generate normalized coordinates in [0, 1]:
  - `rows = torch.arange(grid_h, dtype=torch.float32) / (grid_h - 1)` — values [0, 0.167, ..., 1.0]
  - `cols = torch.arange(grid_w, dtype=torch.float32) / (grid_w - 1)`
- Create grid: `row_coords = rows.repeat_interleave(grid_w)`, `col_coords = cols.repeat(grid_h)`
- Stack: `pos_enc = torch.stack([row_coords, col_coords], dim=1)` — shape [49, 2]
- Concatenate: `torch.cat([x, pos_enc.to(x.device)], dim=1)` — returns [num_nodes, feature_dim + 2]
- Important: use `pos_enc.to(x.device)` to handle any device mismatch.

**`feature_map_to_graph_hybrid(features: torch.Tensor, global_feat: torch.Tensor, label: int, edge_index: torch.Tensor) -> Data`**:
- Same as existing `feature_map_to_graph` but with PE and global_feat.
- Reshape features [C, H, W] to [H*W, C] node features (same as existing).
- Call `add_positional_encoding_2d(x, grid_h=h, grid_w=w)` to get [49, 2050].
- Create Data object with x, edge_index, y.
- CRITICAL: Store global_feat as 2D tensor: `data.global_feat = global_feat.unsqueeze(0)` — shape [1, 2048]. This ensures PyG batching concatenates along dim=0 to produce [batch_size, 2048] instead of flattening to [batch_size * 2048].
- Return the Data object.

**`build_graph_dataset_hybrid(cached_features: List[Dict], edge_index: torch.Tensor) -> List[Data]`**:
- Same pattern as existing `build_graph_dataset` but calls `feature_map_to_graph_hybrid`.
- Each dict must have "features", "global_feat", and "label" keys (from hybrid caching).
- Return list of PyG Data objects.

Important anti-patterns:
- Do NOT apply PE in the model forward pass — it must happen during graph construction (before Data creation).
- Do NOT store global_feat as 1D [2048] — PyG batching will flatten it. Must be [1, 2048].
- Do NOT use raw unnormalized indices [0,1,2,...,6] for PE — normalize to [0, 1] range.
  </action>

**`HybridGraphSAGE(nn.Module)`**:
- Docstring: "Two-layer GraphSAGE with CNN+GNN hybrid fusion classifier."
- Constructor args:
  - `node_dim: int = 2050` (2048 features + 2 PE coords)
  - `hidden_channels: int = 256`
  - `embed_channels: int = 128`
  - `global_channels: int = 2048` (ResNet-50 avgpool dimension)
  - `num_classes: int = NUM_CLASSES` (use existing env-driven NUM_CLASSES)
  - `dropout: float = 0.5`
- Layers:
  - `self.conv1 = SAGEConv(node_dim, hidden_channels)` — node_dim=2050 not 2048
  - `self.conv2 = SAGEConv(hidden_channels, embed_channels)`
  - `self.dropout = nn.Dropout(dropout)`
  - `self.relu = nn.ReLU()`
  - `fusion_dim = embed_channels + global_channels` — 128 + 2048 = 2176
  - `self.classifier = nn.Linear(fusion_dim, num_classes)`

- Forward method signature: `forward(self, x, edge_index, batch, global_feat)`:
  - GNN path: conv1 -> relu -> dropout -> conv2 -> relu -> global_mean_pool(x, batch) -> gnn_embed [batch_size, 128]
  - Fusion: `fused = torch.cat([gnn_embed, global_feat], dim=1)` — [batch_size, 2176]
    NOTE: global_feat arrives as [batch_size, 2048] from PyG batching (already correct shape from the [1, 2048] per-graph storage). No squeeze needed if PyG produces [batch_size, 2048] directly.
    BUT if PyG batching produces [batch_size, 1, 2048] (since per-graph is [1, 2048] and cat_dim=0 adds a batch dim), handle with: `if global_feat.dim() == 3: global_feat = global_feat.squeeze(1)`.
    VERIFY during testing which shape PyG actually produces. The graph.py verify script above tests this — use the same shape handling.
  - Classification: `logits = self.classifier(fused)` — returns [batch_size, num_classes]

- Add `import torch.nn.functional as F` at top of file if not already present (for consistency, though using self.relu is fine too).

Important anti-patterns:
- Do NOT add a projection layer before fusion (e.g., Linear(2048, 256) for CNN features) — unnecessary parameters on small dataset per research recommendation.
- Do NOT change the existing GraphSAGEClassifier class — it must remain for comparison.
- Do NOT hardcode num_classes=16 — use the existing NUM_CLASSES env pattern.
