  <name>Task 1: Create feature extraction and caching module</name>
  <files>src/features.py</files>
  <action>
Create `src/features.py` with the following exports:

**`create_resnet_extractor(config: Config)`**:
- Load ResNet-50 with `weights=ResNet50_Weights.IMAGENET1K_V2` (NOT deprecated `pretrained=True`)
- Use `create_feature_extractor(model, return_nodes={"layer4": "layer4"})` from `torchvision.models.feature_extraction`
- Set model to `.eval()` mode (critical for BatchNorm/Dropout)
- Freeze all parameters: `for param in extractor.parameters(): param.requires_grad = False`
- Move to `config.device`
- Return the feature extractor model
- Print model info: number of parameters (frozen)

**`extract_features_batch(extractor, images: torch.Tensor, device)`**:
- Takes a batch of preprocessed images [B, 3, 224, 224] already as a tensor
- Moves images to device
- Runs through extractor with `torch.no_grad()` context
- Returns layer4 output tensor [B, 2048, 7, 7] moved to CPU (for portability when caching)
- Important: return `.cpu()` tensors so cached files are device-agnostic

**`cache_features(samples, extractor, transform, config: Config, split: str)`**:
- Takes a list of sample dicts (from load_rvl_cdip), the extractor, transform, and config
- Creates cache directory: `config.cache_dir / split / ` (e.g., `./cached_features/train/`)
- Processes samples in batches of `config.batch_size`:
  - Apply transform to each image in the batch
  - Stack into tensor [B, 3, 224, 224]
  - Extract features via extract_features_batch -> [B, 2048, 7, 7]
  - Save each feature individually: `torch.save({"features": features[i], "label": sample["label"]}, path / f"{idx:06d}.pt")`
  - Each .pt file contains a dict with "features" (tensor [2048, 7, 7]) and "label" (int)
- Show progress with tqdm
- Print total cached count and directory path when done
- Return the cache directory path

**`load_cached_features(config: Config, split: str)`**:
- Loads all .pt files from `config.cache_dir / split /`
- Returns list of dicts with "features" (tensor) and "label" (int)
- Sorts files by name to preserve order
- Uses `torch.load(..., weights_only=True)` for security (PyTorch 2.0+ recommendation). Note: since we save dicts with tensor + int, we need `weights_only=False` or save features and labels separately. Use `weights_only=False` with a comment explaining why (dict contains non-tensor int label).
- Print loaded count

Important anti-patterns to avoid:
- Do NOT extract after avgpool -- must be layer4 output preserving 7x7 spatial structure
- Do NOT keep features on GPU when saving -- always .cpu() for portability
- Do NOT load all features into RAM at once for large datasets -- but for Phase 1 (100 samples) this is fine
- Do NOT use deprecated `pretrained=True` -- use `weights=ResNet50_Weights.IMAGENET1K_V2`
  </action>
  <verify>
