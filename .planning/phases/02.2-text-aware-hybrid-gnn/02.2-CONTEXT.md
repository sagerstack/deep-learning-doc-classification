# Phase 2.2: Text-Aware Hybrid GNN - Context

**Gathered:** 2026-03-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Add CRAFT text density features as an additional node feature in the hybrid GNN graph, targeting improved classification on layout-sensitive document types (form, invoice, memo, scientific_report). This phase adds text density extraction to the baseline notebook's caching pipeline and creates a new text-aware hybrid notebook for training and evaluation.

Does NOT include: full OCR, text content extraction, new model architectures beyond adapting the existing hybrid approach, or full-dataset training.

</domain>

<decisions>
## Implementation Decisions

### Text detection approach
- Use CRAFT text detector (pretrained, frozen) to produce character-level text probability heatmaps
- Raw heatmap values as-is — no thresholding, no clamping. Let the GNN learn to weight the signal
- Downsample heatmap to 7x7 using average pooling to match the CNN feature map grid
- Device: MPS on subset for now, scale to full dataset later (same strategy as hybrid notebook)

### Feature integration
- Single float per node: text density appended as 1 dimension → node_dim = 2051 (2048 CNN + 2 PE + 1 text_density)
- Always present in text-aware pipeline (not a toggle). Ablation done by comparing against Phase 2.1 notebook results

### Cache augmentation
- Augment existing .pt files by adding `text_density` key (same pattern as `global_feat` from Phase 2.1)
- Backward compatible: all other notebooks ignore the `text_density` key
- New additive cell in baseline notebook under Section 7 — assumes `data` variable exists from prior cells
- Prerequisites for re-running: only setup + data loading cells (C1-C4 equivalent), NOT training or feature extraction
- Must NOT re-run ResNet-50 feature extraction or model training

### Notebook structure
- New notebook: `rvl-cdip-text-aware-hybrid.ipynb` — clean separation from Phase 2.1 hybrid notebook
- Comparison section shows: CNN baseline (ResNet-50) vs Hybrid GraphSAGE (no text, from Phase 2.1) vs Text-Aware Hybrid
- Per-class F1 breakdown to validate hypothesis that text density helps layout-sensitive document types

### Training approach
- Same hyperparameters as Phase 2.1: hidden=64, embed=32, dropout=0.7, lr=5e-4, label_smoothing=0.1, patience=15, min_epochs=20
- Fair comparison: only variable is the text density feature
- New model class: `TextAwareGraphSAGE` in `model.py` — separate from `HybridGraphSAGE`
- New graph function: `build_graph_dataset_text_aware` in `graph.py` — separate from hybrid functions

### Claude's Discretion
- CRAFT configuration details (link threshold, text threshold)
- Exact cell structure and ordering in baseline notebook
- TextAwareGraphSAGE internal architecture (should mirror HybridGraphSAGE with node_dim=2051)

</decisions>

<specifics>
## Specific Ideas

- Text density targets CNN's weakest classes: form (0.45 F1), scientific_report (0.40), memo (0.49), invoice (0.51)
- The hypothesis: "forms always have alternating text-blank patterns, invoices always have header+table+footer text layout" — text density makes these spatial patterns explicit
- Baseline notebook cell must be self-contained and additive — just augments cache, doesn't touch existing cells

</specifics>

<deferred>
## Deferred Ideas

- Full OCR text content as features — much heavier, could be its own phase
- Learned text embedding (Linear projection of text density) — try if single float doesn't help enough
- Text region-based graph construction (edges between text blocks instead of k-NN grid) — architectural change, separate phase

</deferred>
