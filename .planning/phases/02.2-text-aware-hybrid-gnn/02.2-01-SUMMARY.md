# Phase 2.2 Plan 01: Text-Aware Hybrid GNN — Source Module Summary

**One-liner:** DBNet text density extraction (doctr) with [7,7] heatmap downsampling, text-aware graph construction appending 1 text feature per node (2050 -> 2051), and standalone TextAwareGraphSAGE model.

## What Was Built

Three source module additions forming the complete text-aware pipeline backbone:

**`src/text_features.py`** (new file):
- `create_text_detector(config)`: Factory for frozen doctr `db_mobilenet_v3_large` predictor, with MPS-to-CPU device fallback
- `extract_text_density(image, detector, device)`: Converts PIL image to [7,7] float32 text probability tensor using raw DBNet probability map (sigmoid output before post-processing), downsampled via `adaptive_avg_pool2d`
- `augment_cache_with_text_density(config, split, detector, samples)`: Idempotent cache augmentation — skips files that already have `text_density` key, shows tqdm progress

**`src/graph.py`** (extended):
- `feature_map_to_graph_text_aware`: Builds graph with node_dim=2051 (CNN [2048] + PE [2] + text_density [1]). Reuses `add_positional_encoding_2d`, appends text density last
- `build_graph_dataset_text_aware`: List builder with early ValueError if `text_density` key missing from cache

**`src/model.py`** (extended):
- `TextAwareGraphSAGE`: Independent class mirroring HybridGraphSAGE with node_dim=2051 default. 2-layer SAGEConv, BatchNorm before fusion, concat with global_feat, linear classifier

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Density extraction method | Raw DBNet probability map (`return_model_output=True`) | Richer continuous signal vs binary bounding box coverage |
| Device handling | MPS falls back to CPU for doctr | doctr lacks MPS support; one-time operation so CPU cost acceptable |
| Model structure | Separate class, not subclass | Independent evolution preferred; avoids inheritance fragility |

**2. [Rule 2 - Missing Critical] BatchNorm requires batch_size > 1 in training mode**

- **Found during:** Task 2 verification
- **Issue:** The plan's verify script calls `model(...)` without `model.eval()`. BatchNorm fails with a single sample in training mode.

- **Impact:** Plan verify script needed `model.eval()` addition to work with batch_size=1
