  <files>pyproject.toml, poetry.lock, src/text_features.py</files>
  <action>
IMPORTANT DISCOVERY: The `craft-text-detector` package specified in CONTEXT.md is INCOMPATIBLE with Python 3.13 (archived, pins opencv-python<4.5.4.62 requiring numpy==1.21.2). Use `python-doctr[torch]` instead — it provides an equivalent DBNet text detector that produces text probability heatmaps, is actively maintained, and works on Python 3.13.

1. Add `python-doctr[torch]` as a dependency using `poetry add "python-doctr[torch]"`.

2. Create `src/text_features.py` with three functions:

   **`extract_text_density(image, detector, device) -> torch.Tensor`**
   - Takes a PIL Image (grayscale or RGB), a doctr detection model, and device
   - Converts image to numpy array (RGB, uint8) suitable for doctr
   - Runs through the detector model to get the text probability/heatmap output
   - The doctr detector internally produces a probability map. Access it by calling the model directly:
     ```python
     from doctr.models import detection_predictor
     detector = detection_predictor(arch='db_mobilenet_v3_large', pretrained=True, assume_straight_pages=True)
     ```
     Then use the model's underlying detection architecture to get the raw heatmap before post-processing. The predictor's `det_predictor.model` can be called directly on a batch tensor to get the probability map.
   - Alternative simpler approach: Use the detector normally and reconstruct density from detected word bounding boxes — for each 7x7 grid cell, count the fraction of area covered by detected text boxes. This avoids needing to hack into model internals.
   - Downsample the resulting heatmap to [7, 7] using `torch.nn.functional.adaptive_avg_pool2d`
   - Return tensor of shape [7, 7] (float32, values in ~[0, 1] range)
   - Use `db_mobilenet_v3_large` arch (lightweight, fast on MPS) NOT `db_resnet50` (too heavy for per-image extraction)

   **`augment_cache_with_text_density(config, split, detector, samples) -> None`**
   - `samples`: list of dicts, each with an `"image"` key containing a PIL Image (matches HuggingFace dataset row format). Indexed same order as cached .pt files (000000.pt, 000001.pt, ...).
   - Loads each cached .pt file from `config.cache_dir / split`
   - If `text_density` key already exists, skip that file
   - Otherwise: get the PIL image from `samples[i]["image"]`, extract text density via `extract_text_density`, add `text_density` key to the dict, re-save the .pt file
   - Show progress with tqdm

   **`create_text_detector(config) -> detector`**
   - Factory function to create and configure the doctr detector
   - Set pretrained=True, assume_straight_pages=True
   - Move to config.device
   - Freeze all parameters
   - Return the detector in eval mode

  Use the bounding-box approach (reconstruct density from detected boxes) if accessing raw heatmaps from doctr proves complex. The bounding-box approach is:
  - Run detector on image -> get list of word bounding boxes (normalized coords)
  - Create a [H, W] density map initialized to zeros
  - For each bounding box, set the covered region to 1.0
  - Downsample to [7, 7] with adaptive_avg_pool2d
  - Result: each grid cell has a value in [0, 1] representing fraction of text coverage

  Handle device placement: doctr models work on CPU/CUDA. For MPS, fall back to CPU for text detection (doctr may not support MPS). Keep the detection on CPU if MPS causes issues — this is a one-time cache augmentation step, not in the training loop.
  </action>

   **`feature_map_to_graph_text_aware(features, global_feat, text_density, label, edge_index) -> Data`**
   - Same as `feature_map_to_graph_hybrid` but also appends text_density as a node feature
   - Takes text_density [7, 7] tensor
   - Flatten text_density to [49, 1] (one value per node)
   - Node features: CNN features [49, 2048] + PE [49, 2] + text_density [49, 1] = [49, 2051]
   - Apply positional encoding first (reuse `add_positional_encoding_2d`), then concatenate text_density
   - Store global_feat as [1, 2048] (same pattern as hybrid)
   - Return PyG Data with x=[49, 2051], edge_index, y, global_feat

   **`build_graph_dataset_text_aware(cached_features, edge_index) -> List[Data]`**
   - Same pattern as `build_graph_dataset_hybrid` but expects `text_density` key in each cached item
   - Calls `feature_map_to_graph_text_aware` for each item
   - If a cached item is missing `text_density` key, raise ValueError with clear message

2. Add to `src/model.py`:

   **`TextAwareGraphSAGE`** class — separate from HybridGraphSAGE as specified in CONTEXT.md
   - Identical architecture to HybridGraphSAGE but with node_dim=2051 as default
   - Same: 2-layer SAGEConv, dropout, BatchNorm before fusion, concat with global_feat, linear classifier
   - Constructor args: node_dim=2051, hidden_channels=256, embed_channels=128, global_channels=2048, num_classes=NUM_CLASSES, dropout=0.5
   - Forward signature identical to HybridGraphSAGE: forward(x, edge_index, batch, global_feat)
   - The ONLY difference from HybridGraphSAGE is the default node_dim (2051 vs 2050)
   - Keep it as a separate class (not subclass) for clarity and independent evolution
